# UX Refactor & Contextual Extensions Implementation Plan

## Executive Summary
This document outlines the architectural strategy for integrating deeper human-agent interaction within the AI Codex and FinTrader environments. The goal is to eliminate UX friction by allowing contextual Q&A within full-screen tools (like the Trading Chart Modal) and providing a persistent side-by-side workspace for dynamic tools rendered alongside the standard chat feed.

## 1. Modal Agent Context Chat Integration
**Objective:** Allow users to ask the AI Agent questions directly from within the Global Chart Module without closing it, enabling instant insights on streaming data (e.g., "Explain this liquidity grab log").

*   **Location:** Bottom half of the `AnalystSidebar.tsx`.
*   **Architecture:**
    *   Create a miniaturized chat interface (`MiniContextChat.tsx`).
    *   **Context Passing:** The component will intercept the user's message and append a hidden system payload containing the active chart's symbol, current price, and the last 3 analyst logs.
    *   **State Management:** The chat messages will either be isolated to a "Chart Session" or actively mirrored back into the main `MessageList` so the history isn't lost when the modal closes.
*   **UI/UX:** A sticky input field at the bottom of the sidebar, with agent responses appearing as compact, markdown-rendered bubbles inline with the streaming logs.

## 2. Main Chat Stream: Side-by-Side Tools Layout
**Objective:** Move away from a single-column chat feed. Provide a dedicated space for "Interactive Tools" (e.g., charts, execute buttons, tables) generated by the agent, so they don't scroll away when new chat messages arrive.

*   **Location:** `client/src/pages/Chat.tsx` and `client/src/components/chat/MessageList.tsx`.
*   **Architecture:**
    *   **Layout Shift:** On desktop (`lg` and above), split the main view into a CSS Grid (e.g., `grid-cols-[1fr_400px]` or `w-[65%] / w-[35%]`).
    *   **Left Column (Chat Stream):** Houses the standard `MessageList` and `ChatInput`. Focuses purely on dialogue.
    *   **Right Column (Active Tool Harness):** A new sticky column `ContextualToolSidebar.tsx`.
    *   **Tool Rendering Logic:** When the AI dispatches a custom UI block (like a manual buy/sell trigger or a data table), instead of rendering it inline within the chat bubble, the message component will dispatch the tool to a global state (e.g., `activeContextTool`). The Right Column listens to this state and renders the tool.
*   **Mobile Fallback:** On small screens, the side-by-side view collapses, and active tools render either inline (legacy style) or via an expandable bottom-sheet.

## 3. Rollout Phases
*   **Phase A:** Build the `MiniContextChat` and wire it into the `AnalystSidebar`. Ensure it can send messages to the backend and display responses.
*   **Phase B:** Refactor the main `Chat.tsx` layout to establish the side-by-side structure.
*   **Phase C:** Implement the global state routing to project agent-generated UI components from the chat history into the new Tools column.
